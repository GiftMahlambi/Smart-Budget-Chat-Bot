Diagnosis
- Relevant log lines: the build command invoked here [line 66](#L66) and Vite start here [line 68](#L68). Vite fails to load the config [line 69](#L69) and the error message is "PORT environment variable is required but was not provided." [line 71](#L71). The thrown error originates from the bundled vite.config.ts [line 72](#L72). The script exits with failure [line 81](#L81)-[line 83](#L83).
- Error type: runtime error thrown while loading vite.config.ts during the build.
- Cause: vite.config.ts expects a PORT environment variable to be present and explicitly throws when it is not provided. That makes the Vite config load fail in the Netlify build environment where PORT is not set.

Solution
1) Quick fix (set PORT in Netlify build environment)
- Add a PORT environment variable in your Netlify site settings (or in netlify.toml) so the build sees it.
  - Netlify UI: Site > Settings > Build & deploy > Environment > Add variable: key = PORT, value = 5173 (or whichever port your config expects).
  - Or add to netlify.toml at the repo root:
```toml
[build.environment]
  PORT = "5173"
```
- Re-run the deploy.

2) Recommended fix (make vite.config.ts robust so builds don't require PORT)
- Modify artifacts/mockup-sandbox/vite.config.ts to not require PORT during a production build. Only use PORT for dev/serve, or provide a sensible default instead of throwing. Example change:
```ts
// TypeScript (vite.config.ts)
import { defineConfig } from 'vite';

const port = process.env.PORT ?? '5173';

export default defineConfig(({ command }) => {
  if (command === 'serve') {
    // only require/use PORT for local dev server
    return {
      server: {
        port: Number(port),
      },
    };
  }

  // For build, do not require PORT — return your build config
  return {
    // ...your build config
  };
});
```
- Alternatively, replace the explicit throw in vite.config.ts with a default or conditional behavior so the build process can proceed without a PORT env var.

Why this works
- The current build fails while loading vite.config.ts because it throws if PORT is missing ([line 69](#L69) → [line 71](#L71)). Either supplying PORT in the build environment or making vite.config.ts tolerant of a missing PORT prevents the thrown error and allows Vite to load and run the build.

- artifacts/mockup-sandbox/vite.config.ts
